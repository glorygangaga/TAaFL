# Отличия компилятора от интерпретатора

<hr>

__Компилятор__ и __Интерпетатор__ - это инструменты для преобразования кода из высокого уровня (понятному человеку) в низкоуровневые конструкции, которые будут понятны процессору. Они выполняют одну задачу, но совершенно разными способами
- __Компилятор__  
> _Переводит весь исходный код программы целиком в машинный или промежуточный код до её запуска._

  Особенности:
  - После компиляции получается исполняемый файл, который можно запустить без участия компилятора.
  - Некоторые компиляторы проверяют код на наличие синтаксических и семантических ошибок до запуска программы, что позволяет их избежать.
  - Идеален для проектов, где требуется производительность
    
  Примеры языков программирования: __C/C++, Rust, Go__  


- __Интерпретатор__  
>_Выполняет программу построчно, сразу анализируя и исполняя команды без предварительного преобразования исходного кода в машинный или байт-код._


  Некоторые особенности: 
  - Каждая строка исходного кода считывается, анализируется и тут же исполняется.
  - Сразу обрабатывает каждую инструкцию, без создания исполняемого файла.
  - Позволяет совершать быстрое тестирование и отладку, тк в программе сразу видны все изменения.  
 
Примеры языков программирования: __Python, JavaScript, Lisp.__

# В чём преимущества компилятора перед интерпретатором?

| Свойство              | Компилятор                                   | Интерпретатор                           |
|-----------------------|----------------------------------------------|-----------------------------------------|
| Скорость выполнения   | Высокая: программа заранее переведена в машинный код | Медленнее: тратится время на построчный анализ и перевод во время выполнения |
| Оптимизация           | Компилятор может оптимизировать код | Из-за построчного выполнения оптимизаций крайне мало         |
| Запуск программы      | Нужен только исполняемый файл | Необходим исходный код |
| Ошибки                | Программа не скомпилируется, пока не будут устранены ошибки            | Ошибки могут появится во время выполнения программы |

# Чем отличаются компилятор в машинный код и в байткод виртуальной машины?

__Компилятор__ - это инструмент, который напрямую переводит исходный код в __машинный код__, понятный процессору. Результатом является исполняемый файл, полностью готовый к запуску на конкретной архитектуре.  

__Компилятор в байткод виртуальной машины__ — это инструмент, который переводит исходный код в промежуточный код (байткод), который уже может выполниться на виртуальной машине.

> __Байткод__ — это компактное промежуточное представление исходного кода, которое выполняется виртуальной машиной (JVM, CLR и другие).


| Свойство              | Компилятор в машинный код                        | Компилятор в байткод виртуальной машины   |
|-----------------------|--------------------------------------------------|-------------------------------------------|
| Целевая среда         | Генерирует код под конкретную архитектуру процессора и ОС     | Генерирует код для виртуальной машины (JVM, CLR и другие) |
| Портативность         | __Низкая__: нужно перекомпилировать под каждую платформу | __Высокая__: один байткод работает на разных системах при наличии виртуальной машины для данного байткода |
| Скорость выполнения   | __Максимально высокая__: код исполняется напрямую процессором   | __Низкая__: требуется интерпретация или JIT-компиляция |
| Размер итогового файла| __Большой__: машинный код специфичен | __Меньше__: байткод более компактная версия исходного кода программы   |

<hr>

> __Поток токенов__ — последовательность ключевых слов, идентификаторов и операторов, на которые разбивается исходный код.

>__AST (Abstract Syntax Tree)__ — дерево, отражающее синтаксическую структуру программы(вложенность выражений, приоритет операторов).

>__Аннотированный AST__ — AST с дополнительной информацией (типы данных, информация о памяти и регистрах).

>__IR (Intermediate Representation, напр. LLVM IR)__ — промежуточное представление программы, используемое для анализа и оптимизации перед генерацией кода.

>__Оптимизированный IR__ — улучшенная версия IR, прошедшая оптимизации для ускорения работы и уменьшения объёма кода.

>__Машинный код / байткод__ — конечный результат компиляции, выполняемый процессором или виртуальной машиной.
